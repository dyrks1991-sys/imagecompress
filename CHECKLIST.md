## Risks Acknowledged ✓

- [x] **R1:** Large images (> 20MB) freeze the UI thread → Mitigation: browser-image-compression runs in a Web Worker by default. Cap upload at 50MB with clear error message.
- [x] **R2:** PNG output via Canvas not smaller than input → Mitigation: For PNG output, warn user: "PNG compression is lossless — file size may not reduce. Use JPEG or WebP for size reduction."
- [x] **R3:** Safari CORS on canvas.toBlob() → Mitigation: Only process same-origin files (user-uploaded). Test on Safari 16+ before release.
- [x] **R4:** Memory leak from unreleased Object URLs → Mitigation: Call URL.revokeObjectURL() in useEffect cleanup whenever compressedUrl or originalUrl changes.
- [x] **R5:** browser-image-compression v2 API change → Mitigation: Pin to exact version in package.json. Wrap in compress.ts to isolate API surface.
